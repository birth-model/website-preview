üìã Overview

This guide covers the deployment and configuration of the Birth Model iOS application for enterprise environments. Birth Model integrates with your existing MDM solution and supports Apple Business Manager for streamlined deployment.

Platform Support

iOS: Apple Business Manager + MDM (Intune, Jamf, Kandji, Workspace ONE, Mosyle)

Deployment Options

  • Apple Business Manager: Volume purchase and automatic deployment via MDM
  • Managed App Configuration: Customize branding and settings per hospital
  • Shared Device Mode: Multi-user support for shared L&D workstations

üçé Apple Business Manager

Deploy Birth Model through Apple Business Manager (ABM) for seamless enterprise distribution without requiring individual Apple IDs on each device.

Benefits

  • Volume Purchase Program (VPP): Acquire app licenses centrally
  • Automated Device Enrollment: Pre-configure devices before deployment
  • Silent App Installation: Deploy automatically without user intervention
  • License Management: Reassign licenses when devices are retired

Setup Steps

Sign in to Apple Business Manager

Navigate to business.apple.com with your organization's admin credentials.

Acquire Birth Model Licenses

Go to Apps and Books, search for "Birth Model", and acquire the required number of licenses for your organization.

Assign to MDM Server

Assign the licenses to your MDM server location. The app will sync automatically.

Configure in MDM

Create an app deployment policy in your MDM portal and apply managed app configuration (see below).

üë• Shared Device Mode

Birth Model supports Microsoft Entra ID Shared Device Mode for devices used by multiple clinicians across shifts. This is ideal for shared workstations, nursing stations, and mobile carts in Labor & Delivery units.

Benefits

  • Single Sign-Out: When a clinician signs out, all app data and tokens are cleared automatically
  • Fast User Switching: Next user can sign in immediately with their credentials
  • HIPAA Compliance: Ensures no PHI persists between user sessions
  • Reduced Device Inventory: Multiple clinicians can share fewer devices securely

Prerequisites

  • Devices must be enrolled in your MDM with Supervised mode enabled
  • Microsoft Authenticator app must be installed and configured as the broker
  • Azure AD / Entra ID tenant must have Shared Device Mode enabled
  • Birth Model app installed via MDM (Apple Business Manager)

How It Works

Device Registration

Microsoft Authenticator registers the device as a "shared device" with your Entra ID tenant.

Clinician Sign-In

Clinician signs into Birth Model using their hospital credentials. The app detects Shared Device Mode automatically.

Shift Work

Clinician uses Birth Model normally throughout their shift with full access to patient data.

Global Sign-Out

When the clinician signs out, all tokens and cached data are cleared from the device across all participating apps.

Issue Resolution
App not appearing on device Verify ABM sync with MDM; check device group assignment; ensure license is available
Configuration not applied Ensure managed app config is linked to correct app version; force-quit and relaunch app
Background image not loading Verify URL is HTTPS and publicly accessible; check image format and file size
